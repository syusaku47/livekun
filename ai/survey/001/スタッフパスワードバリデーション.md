# スタッフパスワードバリデーション

## 概要

PPM Cloudのスタッフパスワードに関するバリデーションルールをまとめた資料

## バリデーションルール

### パスワードの要件

- **文字数**: 8文字以上15文字以下
- **使用可能文字**: 半角ASCII文字（スペースから~まで、コード範囲: 0x20-0x7E）
- **複雑性要件**: 以下の3種類を必ず含む必要がある
  - 英小文字（a-z）
  - 英大文字（A-Z）
  - 数字または記号（0-9!@#$%^&*()_+）

### 正規表現

```regex
/^(?=.*[a-z])(?=.*[A-Z])(?=.*[0-9!@#$%^&*()_+])[ -~]{8,15}$/
```

#### 正規表現の解説

- `(?=.*[a-z])`: 英小文字を1文字以上含む（先読みアサーション）
- `(?=.*[A-Z])`: 英大文字を1文字以上含む（先読みアサーション）
- `(?=.*[0-9!@#$%^&*()_+])`: 数字または記号を1文字以上含む（先読みアサーション）
- `[ -~]{8,15}`: 半角ASCII文字（スペース～チルダ）で8文字以上15文字以下

### バリデーションの実装箇所

#### 1. スタッフ新規登録

- **ファイル**: `server/app/Http/Requests/StaffStoreRequest.php`
- **クラス**: `StaffStoreRequest`
- **メソッド**: `rules()`
- **行数**: 26-36

**バリデーションルール**:
```php
'password' => [
    'required',
    'string',
    'regex:/^(?=.*[a-z])(?=.*[A-Z])(?=.*[0-9!@#$%^&*()_+])[ -~]{8,15}$/'
]
```

**エラーメッセージ**:
- `password.regex`: 「パスワードは英小文字と英大文字かつ数字もしくは記号を含む8文字以上15文字までの形で指定してください」

**関連コントローラー**:
- `server/app/Http/Controllers/StaffController.php::store()`

#### 2. スタッフパスワード変更

- **ファイル**: `server/app/Http/Requests/StaffUpdateRequest.php`
- **クラス**: `StaffUpdateRequest`
- **メソッド**: `rules()`
- **行数**: 25-43

**バリデーションルール**:
```php
// パスワード更新時のみ
if ($request->filled('password')) {
    'password' => [
        'required',
        'string',
        'confirmed',
        'regex:/^(?=.*[a-z])(?=.*[A-Z])(?=.*[0-9!@#$%^&*()_+])[ -~]{8,15}$/'
    ]
}
```

**エラーメッセージ**:
- `password.regex`: 「パスワードは英小文字と英大文字かつ数字もしくは記号を含む8文字以上15文字までの形で指定してください」

**関連コントローラー**:
- `server/app/Http/Controllers/StaffController.php::update()`

#### 3. パスワードリセット（メール認証）

- **ファイル**: `server/app/Http/Controllers/ResetPasswordController.php`
- **メソッド**: `chgpassword()`
- **行数**: 72-75

**バリデーションルール**:
```php
// スタッフの場合（userType == UserConst::STAFF (1)）
$validate['password'] = [
    'required',
    'string',
    'confirmed',
    'regex:/^(?=.*[a-z])(?=.*[A-Z])(?=.*[0-9!@#$%^&*()_+])[ -~]{8,15}$/'

```

**エラーメッセージ**:
- `password.regex`: 「パスワードは英小文字と英大文字かつ数字もしくは記号を含む8文字以上15文字までの形で指定してください」

**条件**: `userType == UserConst::STAFF (1)` の場合に適用

## パスワード処理の流れ

### 1. スタッフ新規登録フロー

1. 店舗管理者がスタッフ情報を入力（メールアドレス、スタッフID、名前、パスワード、権限）
2. `StaffStoreRequest` でバリデーション
3. パスワードはハッシュ化して保存（`ModelBase::getHashPassword()`）
4. DynamoDBへスタッフ情報を保存
5. スタッフへ登録完了メールを送信

### 2. スタッフパスワード変更フロー

1. ログイン後の設定画面からパスワード変更
2. 新パスワード + 確認用パスワード入力
3. `StaffUpdateRequest` でバリデーション
4. `Staff::changePassword()` 実行
5. パスワードをハッシュ化
6. DynamoDBへパスワード更新

### 3. パスワードリセットフロー（パスワード忘れ）

1. パスワードリセット申請（店舗コード + スタッフID）
2. 登録メールアドレスへリセットURLを送信
3. URLからパスワードリセット画面にアクセス
4. 新パスワード + 確認用パスワード入力
5. `ResetPasswordController::chgpassword()` でバリデーション
6. トークン検証後、パスワード更新
7. パスワードをハッシュ化してDynamoDBへ更新
8. パスワード変更完了メールを送信

## パスワードのハッシュ化

- **メソッド**: `ModelBase::getHashPassword()`
- スタッフのパスワードは保存時にハッシュ化される
- 店舗パスワードとは異なり、センターサーバーへの同期は不要

## データベース

- **テーブル**: `staff`
- **カラム**: `password`
- **データベース種別**: DynamoDB

## 店舗パスワードとの比較

| 項目 | スタッフパスワード | 店舗パスワード |
|------|-------------------|---------------|
| 文字数 | 8-15文字 | 7文字（固定） |
| 文字種 | 半角ASCII（スペース～チルダ） | 英数字のみ（A-Z, a-z, 0-9） |
| 必須要件 | 英小文字 + 英大文字 + (数字 or 記号) | (英小文字 + 英大文字) or (英小文字 + 数字) or (英大文字 + 数字) |
| 記号使用 | 可能 | 不可 |
| ハッシュ化 | あり | なし（平文） |
| センター同期 | 不要 | 必要 |

## 注意事項

1. スタッフパスワードは店舗パスワードよりも厳格なルールが適用される
2. パスワードはハッシュ化されて保存される
3. 店舗パスワードと異なり、センターサーバーへの同期は不要
4. パスワード変更時は確認用パスワードの入力が必須
