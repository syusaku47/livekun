## 報告

### 調査概要

PPM_CLOUD-4271「【開発・運用規定】PPMアカウントのパスワード規則の変更」に関連して、stagingブランチにおける現状のパスワードバリデーションルールを調査しました。

### 調査結果

#### 1. 店舗パスワードのバリデーション

**現状のルール:**
- 文字数: 7文字（固定）
- 使用可能文字: 英数字のみ（A-Z, a-z, 0-9）
- 複雑性要件: 以下のいずれかの組み合わせを含む
  - 英小文字 + 英大文字
  - 英小文字 + 数字
  - 英大文字 + 数字
- 正規表現: `/^((?=.*[a-z])(?=.*[A-Z])|(?=.*[a-z])(?=.*[0-9])|(?=.*[A-Z])(?=.*[0-9]))[A-Za-z0-9]{7}$/`

**実装箇所:**
1. `server/app/Http/Requests/ShopStorePasswordRequest.php` - 新規登録時
2. `server/app/Http/Requests/ShopChangePasswordRequest.php` - パスワード変更時
3. `server/app/Http/Controllers/ResetPasswordController.php` - パスワードリセット時

**特記事項:**
- パスワードは平文で保存
- センターサーバーとの同期が必要
- センターサーバーへの同期が失敗した場合はロールバック処理を実施

#### 2. スタッフパスワードのバリデーション

**現状のルール:**
- 文字数: 8文字以上15文字以下
- 使用可能文字: 半角ASCII文字（スペース～チルダ、0x20-0x7E）
- 複雑性要件: 以下の3種類を必ず含む
  - 英小文字（a-z）
  - 英大文字（A-Z）
  - 数字または記号（0-9!@#$%^&*()_+）
- 正規表現: `/^(?=.*[a-z])(?=.*[A-Z])(?=.*[0-9!@#$%^&*()_+])[ -~]{8,15}$/`

**実装箇所:**
1. `server/app/Http/Requests/StaffStoreRequest.php` - 新規登録時
2. `server/app/Http/Requests/StaffUpdateRequest.php` - パスワード変更時
3. `server/app/Http/Controllers/ResetPasswordController.php` - パスワードリセット時

**特記事項:**
- パスワードはハッシュ化して保存（`ModelBase::getHashPassword()`）
- センターサーバーへの同期は不要

#### 3. 店舗とスタッフのパスワードルール比較

| 項目 | 店舗パスワード | スタッフパスワード |
|------|---------------|-------------------|
| 文字数 | 7文字（固定） | 8-15文字 |
| 文字種 | 英数字のみ | 半角ASCII全般 |
| 記号使用 | 不可 | 可能 |
| 複雑性 | 2種類の組み合わせ | 3種類必須 |
| ハッシュ化 | なし（平文） | あり |
| センター同期 | 必要 | 不要 |

### 発見事項・注意点

1. **店舗パスワードの制約が厳しい**
   - 7文字固定という制約があり、セキュリティ強度が低い可能性がある
   - 記号が使用できないため、パスワードの複雑性に制限がある

2. **センターサーバーとの同期**
   - 店舗パスワードはセンターサーバーとの同期が必要
   - 同期処理の失敗時にロールバック処理が実装されている

3. **パスワード保存方法の違い**
   - 店舗パスワード: 平文で保存（センター連携のため）
   - スタッフパスワード: ハッシュ化して保存（セキュアな実装）

4. **バリデーション実装の一貫性**
   - 3箇所で同じバリデーションルールが実装されている
   - Request クラスでの実装と Controller での実装が混在

### 推奨事項（パスワード規則変更時の考慮点）

1. **店舗パスワードの強化を検討**
   - 文字数制限の見直し（7文字固定 → 8文字以上など）
   - 記号の使用を許可
   - センターサーバー側の仕様確認が必要

2. **バリデーションの一元管理**
   - バリデーションルールを定数化またはコンフィグ化
   - 重複コードの削減

3. **セキュリティ向上**
   - 可能であれば店舗パスワードもハッシュ化を検討
   - センター連携の仕組みの見直し
