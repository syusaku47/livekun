# 店舗パスワードバリデーション

## 概要

PPM Cloudの店舗パスワードに関するバリデーションルールをまとめた資料

## バリデーションルール

### パスワードの要件

- **文字数**: 7文字（固定）
- **文字種**: 英数字のみ（A-Z, a-z, 0-9）
- **複雑性要件**: 以下のいずれかの組み合わせを含む必要がある
  - 英小文字 + 英大文字
  - 英小文字 + 数字
  - 英大文字 + 数字

### 正規表現

```regex
/^((?=.*[a-z])(?=.*[A-Z])|(?=.*[a-z])(?=.*[0-9])|(?=.*[A-Z])(?=.*[0-9]))[A-Za-z0-9]{7}$/
```

### バリデーションの実装箇所

#### 1. 店舗パスワード新規登録（スタッフ認証ON時）

- **ファイル**: `server/app/Http/Requests/ShopStorePasswordRequest.php`
- **クラス**: `ShopStorePasswordRequest`
- **メソッド**: `rules()`
- **行数**: 61-69

**バリデーションルール**:
```php
'password' => [
    'required',
    'string',
    'confirmed',
    'regex:/^((?=.*[a-z])(?=.*[A-Z])|(?=.*[a-z])(?=.*[0-9])|(?=.*[A-Z])(?=.*[0-9]))[A-Za-z0-9]{7}$/'
]
```

**エラーメッセージ**:
- `password.regex`: 「パスワードは7文字かつ英語小文字、英語大文字、数字の2種類を含む形で指定してください」

**関連コントローラー**:
- `server/app/Http/Controllers/ShopSendMailController.php::store_shop_password()`

## パスワード処理の流れ

### 1. 店舗パスワード新規登録フロー

1. 店舗登録時にメール送信（`ShopController::store()`）
2. メール内のURLからパスワード設定画面にアクセス
3. パスワード入力（新規パスワード + 確認用）
4. `ShopStorePasswordRequest` でバリデーション
5. DynamoDBへパスワード保存
6. センターサーバーへパスワード変更リクエスト（`ShopService::changeCenterShopPassword()`）

### 2. 店舗パスワード変更フロー

1. ログイン後の設定画面からパスワード変更
2. 旧パスワード + 新パスワード + 確認用パスワード入力
3. `ShopChangePasswordRequest` でバリデーション
4. `Shop::changePassword()` 実行
5. DynamoDBへパスワード更新
6. センターサーバーへパスワード変更リクエスト

### 3. パスワードリセットフロー（パスワード忘れ）

1. パスワードリセット申請（`ShopSendMailController::reset_password()`）
2. 登録メールアドレスへリセットURLを送信
3. URLからパスワードリセット画面にアクセス
4. 新パスワード + 確認用パスワード入力
5. `ResetPasswordController::chgpassword()` でバリデーション
6. トークン検証後、パスワード更新
7. DynamoDBへパスワード更新
8. センターサーバーへパスワード変更リクエスト

## データベース

- **テーブル**: `shops`
- **カラム**: `password`
- **データベース種別**: DynamoDB

## 注意事項

1. パスワードはセンターサーバーとPPM Cloudの両方で管理される
2. パスワード変更時はDynamoDB更新後にセンターサーバーへの同期が必要
3. センターサーバーへの同期が失敗した場合はロールバック処理が実行される
4. 店舗パスワードとスタッフパスワードでバリデーションルールが異なる
