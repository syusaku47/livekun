# センターログイン処理まとめ

## 概要

このドキュメントは、ppm-cloud-apiにおけるセンターサーバーへのログイン処理をまとめたものです。

## 主要なログイン処理クラス

### 1. CenterServerAccessService (メイン)
**ファイルパス**: `ppm-cloud-api/server/app/Services/CenterServerAccessService.php`

#### 提供するログイン機能

##### 1.1 バックエンドログイン (`login()`)
- **目的**: システム(バックエンド)としてセンターサーバーにログイン
- **エンドポイント**: `/api/v1/loginBackend`
- **認証情報**:
  - `shopId`: 環境変数 `CENTER_SERVER_ID`
  - `password`: 環境変数 `CENTER_SERVER_PASSWORD`
- **使用箇所**: バッチ処理やシステム全体の情報取得
- **実装詳細**:
  ```php
  public function login()
  {
      $url = $centerConfig['url'] . $centerConfig['endpoint']['login'];
      $params = [
          'shopId' => $centerConfig['loginID'],
          'password' => $centerConfig['password'],
      ];
      $headers = [
          'X-lnw-App-Id' => $centerConfig['appId'],
      ];
      $response = Http::asForm()->withHeaders($headers)->post($url, $params);
      // セッションIDを取得して保存
      $this->sessionId = (string)$body['body']['data'][0]['sessionId'];
      $this->shopName = (string)$body['body']['data'][0]['shopName'];
  }
  ```

##### 1.2 店舗ログイン (`shopLogin()`)
- **目的**: 特定の店舗としてセンターサーバーにログイン
- **エンドポイント**: `/api/v1/login`
- **認証情報**:
  - `kijshopCd`: 店舗コード (7桁の数字)
  - `password`: 店舗パスワード
- **使用箇所**: ユーザーログイン、店舗固有の情報取得
- **実装詳細**:
  ```php
  public function shopLogin(string $shopId, string $password)
  {
      $url = $centerConfig['url'] . $centerConfig['endpoint']['shoplogin'];
      $params = [
          'kijshopCd' => $shopId,
          'password' => $password,
      ];
      $headers = [
          'X-lnw-App-Id' => $centerConfig['appId'],
      ];
      $response = Http::asForm()->withHeaders($headers)->post($url, $params);
      // セッションIDを取得して保存
      $this->sessionId = (string)$body['body']['data'][0]['sessionId'];
      $this->shopName = (string)$body['body']['data'][0]['shopName'];
  }
  ```

##### 1.3 ログアウト処理
- **logout()**: バックエンドログアウト
- **shopLogout()**: 店舗ログアウト
- **エンドポイント**: `/api/v1/logout`
- **必須ヘッダー**:
  - `X-Session-Id`: ログイン時に取得したセッションID
  - `X-lnw-App-Id`: アプリケーションID

##### 1.4 ログイン・ログアウトを含む処理実行 (`executeWithLoginLogout()`)
- **目的**: ログイン→処理実行→ログアウトを自動で行う
- **実装詳細**:
  ```php
  public function executeWithLoginLogout(\Closure $process)
  {
      $this->login();
      try {
          $result = $process();
      } finally {
          $this->logout();
      }
      return $result;
  }
  ```

### 2. CloudFolder\CenterServerAccessService
**ファイルパス**: `ppm-cloud-api/server/app/Services/CloudFolder/CenterServerAccessService.php`

- CenterServerAccessServiceを継承
- CloudFolder関連の機能に特化
- 同じログイン処理を実装
- 追加機能:
  - `executeWithLoginLogout(\Closure $process, $kijshopCd)`: 店舗ログイン版
  - `executeWithBackendLoginLogout(\Closure $process)`: バックエンドログイン版

### 3. AbstractCenterServerGateway
**ファイルパス**: `ppm-cloud-api/server/app/Services/Center/AbstractCenterServerGateway.php`

- **目的**: センターサーバーアクセスの基底クラス
- **ログイン処理**: `prepare()` メソッド内で `CenterServerAccessService` の `login()` を呼び出し
- **ログアウト処理**: `after()` メソッド内で `logout()` を呼び出し
- **使用方法**: このクラスを継承したサービスが処理前に `prepare()`、処理後に `after()` を呼び出す

## セッションID管理

### セッションIDの取得
- ログイン成功時にレスポンスボディから取得
- JSON構造: `$body['body']['data'][0]['sessionId']`
- クラスプロパティとして保持: `$this->sessionId`

### セッションIDの使用
- 全てのセンターAPI呼び出しで必要
- HTTPヘッダーに設定: `X-Session-Id: {セッションID}`
- アプリケーションIDも必須: `X-lnw-App-Id: {アプリID}`

### セッションIDの有効期限
- センターサーバー側で管理
- 期限切れの場合、再ログインが必要

## ログイン処理のフロー

### 通常のフロー
```
1. CenterServerAccessServiceインスタンス化
2. login() または shopLogin() 呼び出し
3. センターサーバーからセッションID取得
4. セッションIDを使用してAPI呼び出し
5. 処理完了後、logout() または shopLogout() 呼び出し
```

### executeWithLoginLogout を使用したフロー
```
1. CenterServerAccessServiceインスタンス化
2. executeWithLoginLogout() にクロージャを渡す
3. 自動的にログイン→処理実行→ログアウト
```

## エラーハンドリング

### ログイン失敗時
- `login()` / `shopLogin()` は `false` を返す
- または、エラー情報を含む配列を返す (shopLoginの場合)
- エラーコードが200以外の場合は失敗

### ログアウト失敗時
- `logout()` / `shopLogout()` は `false` を返す
- ログに記録される

### 例外処理
- 通信エラーなどの例外が発生した場合
- ログに記録: `LogGenerator::logAndMailSend()` または `Log::error()`
- `false` を返す

## 設定情報

### 環境変数
- `CENTER_SERVER_HOST`: センターサーバーのURL
- `CENTER_SERVER_ID`: バックエンドログイン用ID
- `CENTER_SERVER_PASSWORD`: バックエンドログイン用パスワード

### 設定ファイル
**ファイルパス**: `ppm-cloud-api/server/config/center.php`

```php
return [
    'url' => env('CENTER_SERVER_HOST', null),
    'loginID' => env('CENTER_SERVER_ID', null),
    'password' => env('CENTER_SERVER_PASSWORD', null),
    'appId' => 2,
    'endpoint' => [
        'login' => '/api/v1/loginBackend',
        'shoplogin' => '/api/v1/login',
        'logout' => '/api/v1/logout',
        'orders' => '/api/v1/orders',
        // その他のエンドポイント...
    ],
];
```

## セキュリティ考慮事項

### IP制限
- センター管理店舗ではグローバルIP制限が実装されている
- `CenterShopIpAddressService` でチェック
- local環境ではスキップ

### パスワード管理
- 環境変数で管理
- ログ出力時はマスク処理

### セッション管理
- セッションIDは `Sessions` テーブルに保存
- トークン (APIキー) とセッションIDを紐付け
- ログアウト時にセッションを削除

## まとめ

ppm-cloud-apiにおけるセンターログイン処理は、主に以下の2つのパターンで実装されています:

1. **バックエンドログイン**: システム全体の管理・バッチ処理用
2. **店舗ログイン**: ユーザー認証・店舗固有の情報取得用

両方とも `CenterServerAccessService` クラスに実装されており、多くのサービスがこのクラスを利用してセンターサーバーと通信しています。

ログイン→処理→ログアウトのフローは、`executeWithLoginLogout()` メソッドを使用することで簡潔に記述できるようになっています。
